## 1. 数学原理

### 1.1 问题定义

给定折线段上**连续三个顶点** P₁、P₂、P₃，判断在 P₂ 处的拐向：
- 从 P₁→P₂ 的行进方向看，到 P₃ 是**左转**还是**右转**还是**直行（共线）**？

### 1.2 向量叉积

构造两个向量：

```
→        →     →
a = P₂ - P₁ = (P₂.x - P₁.x,  P₂.y - P₁.y)

→        →     →
b = P₃ - P₂ = (P₃.x - P₂.x,  P₃.y - P₂.y)
```

二维向量叉积（实际是三维叉积的 z 分量）：

```
            →   →
cross(a, b) = aₓ · b_y - a_y · bₓ

            = (P₂.x - P₁.x) × (P₃.y - P₂.y) - (P₂.y - P₁.y) × (P₃.x - P₂.x)
```

### 1.3 判断规则

| 叉积值 | 含义 | 几何解释 |
|:------:|------|--------|
| `> 0` | **左转** ↰ | b 在 a 的逆时针方向（屏幕坐标系下） |
| `< 0` | **右转** ↱ | b 在 a 的顺时针方向 |
| `= 0` | **共线** → | 三点在同一直线上 |

> ⚠️ **注意坐标系**：Matplotlib 默认 y 轴朝上（数学坐标系），此时 cross > 0 为逆时针（左转）。
> 如果使用屏幕坐标系（y 轴朝下），左右转的判断需要反过来。

### 1.4 公式图解

```
        P₃                      P₁
       ↗                          ↘
      /   cross > 0                 \   cross < 0
     /    左转 ↰                      \  右转 ↱
P₁ → → P₂                    P₃ ← ← P₂

            P₁ → → P₂ → → P₃
                cross = 0
                共线 →
```

### Block 3: Matplotlib 画布嵌入

```python
    def _build_canvas(self):
        """创建 Matplotlib Figure 并嵌入 Tkinter 窗口"""

        # ── 创建中部容器 ──
        # 这个 Frame 水平排列"画布"和"结果面板"
        self.main_frame = tk.Frame(self.root)
        self.main_frame.pack(
            side=tk.TOP,          # 紧接工具栏下方
            fill=tk.BOTH,         # 水平+垂直都填满
            expand=True           # 窗口缩放时，这个区域跟着缩放
        )

        # ── 创建 Matplotlib Figure 和 Axes ──
        self.fig, self.ax = plt.subplots(
            figsize=(7, 5),       # 图形尺寸：7英寸宽 × 5英寸高（英寸，非像素）
            dpi=100               # 每英寸100像素 → 实际 700×500 像素
        )
        self.fig.subplots_adjust(
            left=0.08,            # 左边距占图宽的 8%（给 y 轴标签留空间）
            right=0.95,           # 右边距
            top=0.95,             # 上边距
            bottom=0.08           # 下边距
        )

        # ── 初始化坐标轴 ──
        self.ax.set_xlim(0, 10)   # x 轴范围 0~10（采点时的坐标空间）
        self.ax.set_ylim(0, 10)   # y 轴范围 0~10
        self.ax.set_aspect('equal')  # 等比例坐标轴，确保圆不会变椭圆
        self.ax.grid(True, linestyle='--', alpha=0.3)  # 显示网格线：虚线，30%透明度
        self.ax.set_title("左键采点 | 右键撤销", fontsize=10, color='gray')

        # ── 把 Figure 嵌入 Tkinter ──
        # FigureCanvasTkAgg 是 Matplotlib 与 Tkinter 的桥梁
        self.canvas = FigureCanvasTkAgg(
            self.fig,             # 要嵌入的 Matplotlib Figure 对象
            master=self.main_frame  # 嵌入到哪个 Tkinter 容器中
        )
        # draw() 首次渲染画布
        self.canvas.draw()
        # get_tk_widget() 返回一个 Tkinter Canvas widget，可以用 pack/grid 布局
        self.canvas.get_tk_widget().pack(
            side=tk.LEFT,         # 放在 main_frame 的左侧
            fill=tk.BOTH,         # 填满可用空间
            expand=True           # 窗口缩放时画布跟着缩放
        )
```

**要点**：
- `plt.subplots()` 创建的 Figure 本身是"脱离窗口"的，需要 `FigureCanvasTkAgg` 把它"贴"到 Tkinter 里
- `self.ax.set_aspect('equal')` 很重要——如果 x/y 比例不同，角度判断的可视化会产生误导
- `self.canvas.draw()` 是手动触发重绘，后面每次修改画面都要调一次
- `expand=True` 让画布响应窗口缩放

```python
    def _build_result_panel(self):
        """创建右侧的分析结果显示面板"""

        # ── 结果面板容器 ──
        result_frame = tk.LabelFrame(

            fill=tk.Y,             # 垂直方向填满
            padx=(0, 5),           # 右边留 5px 间距
            pady=5

        # Text 是 Tkinter 的多行文本控件
        self.result_text = tk.Text(
            result_frame,

            font=('Consolas', 10), # 等宽字体，数字对齐好看
            state=tk.DISABLED,     # 初始为"禁用"状态（用户不能手动编辑，只能程序写入）

            wrap=tk.WORD           # 按单词换行（不会把单词从中间切断）
        )
        self.result_text.pack(fill=tk.BOTH, expand=True)

        # ── 为结果文本定义颜色标签 ──
        # Tkinter Text 控件可以给不同文字片段设置不同样式（通过 tag）
        self.result_text.tag_config('left',     foreground='#2E7D32')  # 左转 → 绿色
        self.result_text.tag_config('right',    foreground='#C62828')  # 右转 → 红色
        self.result_text.tag_config('collinear', foreground='#F57F17') # 共线 → 黄色
        self.result_text.tag_config('header',   font=('Microsoft YaHei', 10, 'bold'))  # 标题加粗
```

**要点**：
- `LabelFrame` 比普通 `Frame` 多一个标题边框，视觉上更清晰
- `Text` 控件默认可编辑；设 `state=tk.DISABLED` 防止用户乱改结果，程序写入时要临时设回 `NORMAL`
- `tag_config` 是 Tkinter Text 的精髓——同一个文本框里可以有不同颜色的文字
- 后面写入结果时用 `self.result_text.insert(tk.END, "左转", 'left')` 即可带颜色

### Block 6: 鼠标事件绑定

```python
    def _bindbindevents(self):
        """将 Matplotlib 画布的鼠标事件连接到对应处理方法"""

        # Matplotlib 有自己的事件系统，用 mpl_connect 注册回调函数
        # 注意：这里绑定的是 Matplotlib 的 figure canvas，不是 Tkinter 的

        # 鼠标按下事件 —— 用于采点 和 开始拖动
        self.canvas.mpl_connect(
            'button_press_event',      # Matplotlib 事件名称
            self._on_press              # 回调函数
        )

        # 鼠标移动事件 —— 用于拖动过程中实时更新点的位置
        self.canvas.mpl_connect(
            'motion_notify_event',
            self._on_motion
        )

        # 鼠标释放事件 —— 用于结束拖动
        self.canvas.mpl_connect(
            'button_release_event',
            self._on_release
        )
```

**要点**：
- Matplotlib 的事件系统独立于 Tkinter，有自己的一套事件名
- `button_press_event` 的回调会收到一个 `event` 对象，包含：
  - `event.xdata, event.ydata`：鼠标在**坐标轴数据坐标系**中的位置（不是像素）
  - `event.button`：1=左键, 2=中键, 3=右键
  - `event.inaxes`：鼠标是否在坐标轴区域内（在外面时为 None）
- 拖动功能需要 press + motion + release 三个事件配合

### Block 7: 采点逻辑（左键点击）

```python
    def _on_press(self, event):
        """
        鼠标按下事件的统一入口。
        根据当前模式(self.mode)分发到采点或拖动。
        """

        # ── 前置检查 ──
        if event.inaxes != self.ax:
            # 鼠标不在坐标轴区域内（比如点在了边框上），直接忽略
            return

        if self.mode == 'pick':
            # ── 采点模式 ──
            self._handle_pick(event)

        elif self.mode == 'drag':
            # ── 拖动模式 ──
            self._handle_drag_start(event)

    def _handle_pick(self, event):
        """采点模式下的鼠标按下处理"""

        if event.button == 1:
            # 左键 → 添加新点
            x, y = event.xdata, event.ydata   # 获取点击位置的数据坐标

            self.points.append((x, y))         # 加入点列表
            self.lines_visible = False         # 新增点后，之前的连线作废
            self.analyzed = False              # 分析结果也作废

            self._redraw()                     # 重绘画布（Block 10）
            self._update_status("模式: 采点 | 左键采点，右键撤销")

        elif event.button == 3:
            # 右键 → 撤销最后一个点（Block 8）
            self._handle_undo()
```

**要点**：
- `event.inaxes` 检查很重要，防止在坐标轴外面点击时 `xdata` 为 `None` 导致崩溃
- 每次新增/删除点后都要把 `lines_visible` 和 `analyzed` 重置，保持状态一致
- 左键采点、右键撤销是很直觉的交互设计

### Block 8: 撤销逻辑（右键点击）

```python
    def _handle_undo(self):
        """撤销最后一个采集的点"""

        if not self.points:
            # 点列表已空，没有可撤销的
            self._update_status("没有可撤销的点")
            return

        removed = self.points.pop()            # pop() 移除并返回列表最后一个元素
        self.lines_visible = False
        self.analyzed = False

        self._redraw()                          # 重绘
        self._update_status(
            f"已撤销 ({removed[0]:.1f}, {removed[1]:.1f}) | 右键继续撤销"
        )
```

**要点**：
- `list.pop()` 无参数时移除最后一个元素，时间复杂度 O(1)
- 撤销后要重绘画布，清除已失效的连线和分析标注

### Block 9: 拖动点逻辑（按下-移动-释放）

```python
    def _handle_drag_start(self, event):
        """
        拖动模式下鼠标按下：找到最近的点，开始拖动。
        """
        if event.button != 1 or not self.points:
            return

        # ── 找到离鼠标最近的点 ──
        click = np.array([event.xdata, event.ydata])       # 鼠标位置转 numpy 数组
        pts = np.array(self.points)                         # 所有点转 numpy 二维数组 shape=(n,2)
        distances = np.linalg.norm(pts - click, axis=1)    # 计算欧氏距离，shape=(n,)
        min_idx = np.argmin(distances)                      # 最近点的索引
        min_dist = distances[min_idx]                       # 最近距离

        if min_dist < self.drag_threshold:
            # 距离小于阈值，判定为"选中了这个点"
            self.drag_index = min_idx                       # 记录正在拖动的点的索引
        else:
            # 太远了，没选中任何点
            self.drag_index = None

    def _on_motion(self, event):
        """
        鼠标移动事件：拖动过程中实时更新点的位置。
        """
        # 前置检查：必须在拖动模式 + 已选中某个点 + 鼠标在坐标轴内
        if self.mode != 'drag':
            return
        if self.drag_index is None:
            return
        if event.inaxes != self.ax:
            return

        # ── 更新点坐标 ──
        self.points[self.drag_index] = (event.xdata, event.ydata)

        # ── 实时重绘 ──
        # 拖动时需要保持连线和分析标注（如果之前已经画了的话）
        self._redraw()

    def _on_release(self, event):
        """
        鼠标释放事件：结束拖动。
        """
        if self.mode == 'drag' and self.drag_index is not None:
            self.drag_index = None             # 清除拖动状态
            # 如果之前分析过，拖动后自动重新分析
            if self.analyzed:
                self.analyze_turning()
```

**要点**：
- 拖动功能分三步：press（选中）→ motion（跟随）→ release（松手）
- `np.linalg.norm(pts - click, axis=1)` 是一个**向量化**操作，一次性计算鼠标到所有点的距离，比 for 循环快得多
- `drag_threshold` 控制"吸附灵敏度"——太小会很难选中点，太大会误选
- release 后如果之前分析过，自动重新分析——拖动后结果实时更新，体验好

### Block 10: 画布重绘

```python
    def _redraw(self):
        """
        清空画布并重新绘制所有元素。
        这是画面更新的核心方法，几乎所有操作后都要调用。
        """

        self.ax.cla()  # cla() = clear axes，清空当前坐标轴上的所有内容

        # ── 恢复坐标轴设置（因为 cla() 会把设置也清掉） ──
        self.ax.set_xlim(0, 10)
        self.ax.set_ylim(0, 10)
        self.ax.set_aspect('equal')
        self.ax.grid(True, linestyle='--', alpha=0.3)

        if not self.points:
            # 没有点，显示提示
            self.ax.set_title("左键采点 | 右键撤销", fontsize=10, color='gray')
            self.canvas.draw()     # 触发 Matplotlib 重绘
            return

        # ── 提取所有点的 x, y 坐标 ──
        # zip(*self.points) 把 [(x1,y1),(x2,y2),...] 转置为 [(x1,x2,...), (y1,y2,...)]
        xs, ys = zip(*self.points)

        # ── 画连线（如果已连线） ──
        if self.lines_visible:
            self.ax.plot(
                xs, ys,                # x坐标列表, y坐标列表
                '-',                   # 线型：实线
                color='#1565C0',       # 深蓝色
                linewidth=1.5,         # 线宽 1.5
                zorder=1               # 图层顺序：1（在点的下面）
            )

        # ── 画所有点 ──
        self.ax.scatter(
            xs, ys,
            c='red',                   # 红色点
            s=60,                      # 点大小 60
            zorder=2,                  # 图层 2（在线的上面）
            edgecolors='darkred',      # 点边框深红色
            linewidths=1               # 边框宽度
        )

        # ── 标注点号 ──
        for i, (x, y) in enumerate(self.points):
            self.ax.annotate(
                f'P{i+1}',            # 标注文字：P1, P2, P3, ...
                (x, y),               # 标注目标位置
                textcoords="offset points",  # 偏移单位是"点"（像素级）
                xytext=(8, 8),         # 标注文字相对于目标点偏移 (右8, 上8) 像素
                fontsize=9,
                fontweight='bold',
                color='#333333',       # 深灰色
                zorder=3               # 图层 3（最上层）
            )

        # ── 显示坐标值 ──
        for i, (x, y) in enumerate(self.points):
            self.ax.annotate(
                f'({x:.1f},{y:.1f})',  # 坐标文字，保留 1 位小数
                (x, y),
                textcoords="offset points",
                xytext=(8, -12),       # 在点号下方
                fontsize=7,
                color='#777777'        # 浅灰色，不喧宾夺主
            )

        # ── 更新标题 ──
        self.ax.set_title(
            f"共 {len(self.points)} 个点",
            fontsize=10, color='gray'
        )

        # ── 触发画布重绘 ──
        self.canvas.draw()
```

**要点**：
- 采用**全量重绘**策略（cla → 重新画所有元素），简单可靠，对于几十个点完全没有性能问题
- `zorder` 控制图层叠放顺序：数字越大越靠上。线(1) < 点(2) < 文字(3)
- `zip(*list_of_tuples)` 是 Python 经典的"转置"技巧
- 坐标标注用浅灰色小字，信息丰富但不干扰主视觉

### Block 12: 叉积计算（核心算法）

```python
    @staticmethod
    def cross_product(p1, p2, p3):
        """
        计算三个连续点处的向量叉积，判断拐向。

        参数:
            p1: tuple (x, y) —— 前一个点
            p2: tuple (x, y) —— 当前拐点
            p3: tuple (x, y) —— 后一个点

        返回:
            (cross_value, direction_str)
            cross_value: float，叉积数值
            direction_str: str，"左转 ↰" / "右转 ↱" / "共线 →"

        数学公式:
            →              →
            a = P₂ - P₁    b = P₃ - P₂

                    →   →
            cross = a × b = (ax)(by) - (ay)(bx)

                  = (p2[0]-p1[0]) * (p3[1]-p2[1])
                  - (p2[1]-p1[1]) * (p3[0]-p2[0])
        """

        # 向量 a = P₂ - P₁ 的分量
        ax = p2[0] - p1[0]        # ax = P₂.x - P₁.x
        ay = p2[1] - p1[1]        # ay = P₂.y - P₁.y

        # 向量 b = P₃ - P₂ 的分量
        bx = p3[0] - p2[0]        # bx = P₃.x - P₂.x
        by = p3[1] - p2[1]        # by = P₃.y - P₂.y

        # 二维叉积 = ax*by - ay*bx
        cross = ax * by - ay * bx

        # ── 判断方向 ──
        # 使用一个小的容差值(epsilon)，避免浮点误差导致的误判
        eps = 1e-10               # epsilon: 10的-10次方，极小值

        if cross > eps:
            direction = "左转 ↰"   # 逆时针方向
        elif cross < -eps:
            direction = "右转 ↱"   # 顺时针方向
        else:
            direction = "共线 →"   # 三点共线

        return cross, direction
```

**要点**：
- `@staticmethod` 因为这个方法不需要访问 `self`，它是纯数学计算
- **浮点容差 `eps`**：计算机浮点运算有精度限制，叉积可能不会恰好等于 0，所以用一个极小值判断"近似为零"
- 这就是整个程序**最核心的 4 行代码**：算 ax, ay, bx, by，然后 `ax*by - ay*bx`
- 返回值包含数值和文字，方便后续标注和显示

### Block 13: 拐向分析 + 标注

```python
    def analyze_turning(self):
        """
        对所有内部顶点执行拐向分析，在画布上标注，并在结果面板显示。
        内部顶点 = 除了第一个点和最后一个点之外的所有点。
        """

        if len(self.points) < 3:
            messagebox.showwarning("提示", "至少需要 3 个点才能分析拐向！")
            return

        # ── 确保已连线 ──
        self.lines_visible = True
        self.analyzed = True
        self._redraw()                  # 先重绘基本元素

        # ── 准备结果面板 ──
        self.result_text.config(state=tk.NORMAL)   # 临时解锁文本框，允许写入
        self.result_text.delete('1.0', tk.END)     # 清空旧内容（'1.0'表示第1行第0列）
        self.result_text.insert(tk.END, "═══ 拐向分析结果 ═══\n\n", 'header')

        # ── 定义标注颜色 ──
        color_map = {
            "左转 ↰": '#2E7D32',        # 绿色
            "右转 ↱": '#C62828',        # 红色
            "共线 →": '#F57F17'         # 橙黄色
        }
        tag_map = {
            "左转 ↰": 'left',
            "右转 ↱": 'right',
            "共线 →": 'collinear'
        }

        # ── 遍历每个内部顶点 ──
        # range(1, n-1): 从第2个点到倒数第2个点（索引 1 到 n-2）
        # 因为第一个点没有"前一个点"，最后一个点没有"后一个点"
        n = len(self.points)
        for i in range(1, n - 1):
            p1 = self.points[i - 1]     # 前一个点
            p2 = self.points[i]         # 当前拐点（要判断的点）
            p3 = self.points[i + 1]     # 后一个点

            # ── 调用核心算法 ──
            cross_val, direction = self.cross_product(p1, p2, p3)

            # ── 在画布上标注 ──
            color = color_map[direction]
            self.ax.annotate(
                direction,                     # 标注文字："左转 ↰" / "右转 ↱" / "共线 →"
                (p2[0], p2[1]),                # 标注位置：当前拐点
                textcoords="offset points",
                xytext=(-20, -20),             # 偏移到点的左下方（避免与点号重叠）
                fontsize=10,
                fontweight='bold',
                color=color,                   # 按方向着色
                bbox=dict(                     # 给文字加背景框
                    boxstyle='round,pad=0.3',  # 圆角矩形，内边距 0.3
                    facecolor=color,           # 背景色 = 文字色
                    alpha=0.15                 # 15% 透明度（淡淡的底色）
                ),
                zorder=4                       # 最上层
            )

            # ── 在结果面板写入 ──
            line = f"P{i}→P{i+1}→P{i+2}:  {direction}\n"
            self.result_text.insert(tk.END, line, tag_map[direction])

        # ── 写入叉积数值 ──
        self.result_text.insert(tk.END, "\n═══ 叉积数值 ═══\n\n", 'header')
        for i in range(1, n - 1):
            cross_val, direction = self.cross_product(
                self.points[i-1], self.points[i], self.points[i+1]
            )
            sign = "+" if cross_val >= 0 else ""  # 正数加 + 号，负数自带 - 号
            self.result_text.insert(
                tk.END,
                f"  P{i+1}: {sign}{cross_val:.4f}\n",   # 保留4位小数
                tag_map[direction]                        # 用对应颜色标签
            )

        # ── 锁定结果面板 ──
        self.result_text.config(state=tk.DISABLED)  # 重新禁用编辑

        # ── 刷新画布 ──
        self.canvas.draw()
        self._update_status("分析完成 | 拖动点可实时更新")
```

**要点**：
- `range(1, n-1)` 是关键：只遍历**内部顶点**（有前驱和后继的点）
- Tkinter Text 写入流程：`NORMAL` → 写入 → `DISABLED`，三步不能少
- `bbox` 参数给标注文字加半透明底色，提升可读性
- 叉积数值的正负号带 `+`/`-` 前缀，方便一眼看出方向

### Block 15: 导入坐标文件

```python
    def import_points(self):
        """
        从 CSV/TXT 文件导入坐标点。
        支持格式: 每行一个点，x 和 y 用逗号分隔。
        """

        # ── 弹出文件选择对话框 ──
        filepath = filedialog.askopenfilename(
            title="选择坐标文件",
            filetypes=[
                ("CSV 文件", "*.csv"),              # 下拉菜单第一项
                ("文本文件", "*.txt"),                # 第二项
                ("所有文件", "*.*")                   # 第三项
            ],
            initialdir="."                           # 初始目录：当前目录
        )

        if not filepath:
            return               # 用户点了"取消"

        try:
            new_points = []      # 临时列表，读取成功后再赋值给 self.points

            with open(filepath, 'r', encoding='utf-8') as f:
                reader = csv.reader(f)               # csv.reader 自动按逗号分割每行

                for row_num, row in enumerate(reader, start=1):
                    # row 是一个列表，如 ['1.5', '3.2'] 或 ['x', 'y']（表头）

                    if not row or row[0].strip().startswith('#'):
                        # 跳过空行 和 以 # 开头的注释行
                        continue

                    # ── 尝试解析为浮点数 ──
                    try:
                        x = float(row[0].strip())    # strip() 去除首尾空白
                        y = float(row[1].strip())
                        new_points.append((x, y))
                    except (ValueError, IndexError):
                        # ValueError: 不能转浮点数（可能是表头 "x,y"）
                        # IndexError: 这行不够 2 列
                        # 跳过这些行，不报错（兼容表头）
                        continue

            if not new_points:
                messagebox.showwarning("提示", "文件中没有找到有效坐标！")
                return

            # ── 导入成功，覆盖当前点 ──
            self.points = new_points
            self.lines_visible = False
            self.analyzed = False

            # ── 自动调整坐标轴范围 ──
            xs, ys = zip(*self.points)
            margin = max(
                (max(xs) - min(xs)) * 0.15,    # x 范围的 15% 作为边距
                (max(ys) - min(ys)) * 0.15,    # y 范围的 15%
                0.5                             # 最小边距 0.5，防止所有点在一条线上时没有边距
            )
            self.ax.set_xlim(min(xs) - margin, max(xs) + margin)
            self.ax.set_ylim(min(ys) - margin, max(ys) + margin)

            self._redraw()
            self._update_status(f"已导入 {len(self.points)} 个点 | 来自 {filepath}")
            messagebox.showinfo("导入成功", f"成功导入 {len(self.points)} 个点")

        except Exception as e:
            messagebox.showerror("导入失败", f"读取文件时出错:\n{e}")
```

**要点**：
- `filedialog.askopenfilename` 弹出系统原生文件选择器
- 用 `csv.reader` 而不是手动 `split(',')`，因为 csv.reader 能正确处理引号、转义等边界情况
- **跳过无效行**（注释行、表头）而不是报错，对用户更友好
- 导入后**自动调整坐标轴范围** `set_xlim/set_ylim`，确保所有点可见
- `margin` 取 15% 留白，视觉上不会太挤

```python
            initialfile="turning_result.csv"         # 建议文件名
        )

        if not filepath:
            return

        try:
            with open(filepath, 'w', newline='', encoding='utf-8-sig') as f:
                # newline='': 防止 Windows 下 csv 写入多余空行
                # utf-8-sig: 带 BOM 头的 UTF-8，Excel 打开不会乱码
                writer = csv.writer(f)

                # ── 写入表头 ──
                writer.writerow(['点序号', 'X', 'Y', '拐向', '叉积值'])

                # ── 写入每个点的数据 ──
                n = len(self.points)
                for i, (x, y) in enumerate(self.points):
                    if 0 < i < n - 1:
                        # 内部顶点：有拐向和叉积值
                        cross_val, direction = self.cross_product(
                            self.points[i-1], self.points[i], self.points[i+1]
                        )
                        writer.writerow([
                            f'P{i+1}',              # 点序号
                            f'{x:.4f}',              # X 坐标
                            f'{y:.4f}',              # Y 坐标
                            direction,               # 拐向文字
                            f'{cross_val:.6f}'       # 叉积值（6位小数）
                        ])
                    else:
                        # 首尾点：没有拐向（标记为端点）
                        writer.writerow([
                            f'P{i+1}', f'{x:.4f}', f'{y:.4f}', '端点', '-'
                        ])

            self._update_status(f"已导出到 {filepath}")
            messagebox.showinfo("导出成功", f"结果已保存到:\n{filepath}")

        except Exception as e:
            messagebox.showerror("导出失败", f"保存文件时出错:\n{e}")
```

**要点**：
- `encoding='utf-8-sig'` 是给 Windows Excel 用户的贴心设计，Excel 需要 BOM 头才能正确识别 UTF-8
- `newline=''` 是 Python csv 模块在 Windows 上的固定搭配，否则会多出空行
- 首尾点标记为"端点"，区别于内部拐点

## 4. 坐标文件格式

导入功能支持以下格式的 CSV/TXT 文件：

```csv
# 示例坐标文件（以#开头的行是注释，会被跳过）
x, y
1.0, 2.0
3.5, 5.0
6.0, 3.0
8.0, 7.5
9.5, 4.0
```

**规则**：
- 每行一个点，x 和 y 用**英文逗号**分隔
- 表头行（如 `x, y`）会被自动跳过（无法转为浮点数）
- `#` 开头的行视为注释
- 空行自动跳过

你可以创建一个 `sample_points.csv` 测试文件：

```csv
# 测试数据：Z字形折线
1, 2
4, 8
7, 2
```

## 6. 思考题

完成代码后，尝试思考以下问题：

1. **如果折线是闭合的**（首尾相连），首尾两个点也变成了"内部顶点"，代码该如何修改？
   - 提示：`range` 的范围从 `(1, n-1)` 改为 `(0, n)`，取点时用 `% n` 取模

2. **浮点容差 `eps = 1e-10`** 的取值对结果有什么影响？如果改成 `1e-2` 或 `1e-15` 会怎样？
   - 提示：太大会把"几乎共线但不完全共线"的情况误判为共线；太小会让浮点误差导致的微小叉积被判为有方向

3. **屏幕坐标系**（y 轴朝下）中，叉积的正负与拐向的对应关系要怎么改？
   - 提示：y 轴翻转后，`cross > 0` 变成右转

4. **如何用叉积判断凸多边形？**
   - 提示：如果所有相邻三点的叉积**同号**，则多边形是凸的
