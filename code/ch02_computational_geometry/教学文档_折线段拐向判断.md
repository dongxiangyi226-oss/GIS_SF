## 1. 数学原理

### 1.1 问题定义

给定折线段上**连续三个顶点** P₁、P₂、P₃，判断在 P₂ 处的拐向：
- 从 P₁→P₂ 的行进方向看，到 P₃ 是**左转**还是**右转**还是**直行（共线）**？

### 1.2 向量叉积

构造两个向量：

```
→        →     →
a = P₂ - P₁ = (P₂.x - P₁.x,  P₂.y - P₁.y)

→        →     →
b = P₃ - P₂ = (P₃.x - P₂.x,  P₃.y - P₂.y)
```

二维向量叉积（实际是三维叉积的 z 分量）：

```
            →   →
cross(a, b) = aₓ · b_y - a_y · bₓ

            = (P₂.x - P₁.x) × (P₃.y - P₂.y) - (P₂.y - P₁.y) × (P₃.x - P₂.x)
```

### 1.3 判断规则

| 叉积值 | 含义 | 几何解释 |
|:------:|------|--------|
| `> 0` | **左转** ↰ | b 在 a 的逆时针方向（数学坐标系，y 轴朝上） |
| `< 0` | **右转** ↱ | b 在 a 的顺时针方向 |
| `= 0` | **共线** → | 三点在同一直线上 |

> ⚠️ **注意坐标系**：Matplotlib 默认 y 轴朝上（数学坐标系），此时 cross > 0 为逆时针（左转）。
> 如果使用屏幕坐标系（y 轴朝下），左右转的判断需要反过来。

### 1.4 公式图解

```
        P₃                      P₁
       ↗                          ↘
      /   cross > 0                 \   cross < 0
     /    左转 ↰                      \  右转 ↱
P₁ → → P₂                    P₃ ← ← P₂

            P₁ → → P₂ → → P₃
                cross = 0
                共线 →
```

## 3. 模块详解

---

### 3.1 turning_algo.py — 算法模块

#### 设计目标

这个模块**只做数学计算和文件读写**，不依赖 Tkinter、Matplotlib 等任何 UI 库。
好处：可以在命令行单独测试，也可以被其他程序（比如批量分析脚本）复用。

#### 核心算法：cross_product()

这是整个程序最核心的 4 行数学运算：

```python
ax_ = p2[0] - p1[0];  ay_ = p2[1] - p1[1]   # 向量 a = P2 - P1
bx_ = p3[0] - p2[0];  by_ = p3[1] - p2[1]   # 向量 b = P3 - P2
cross = ax_ * by_ - ay_ * bx_                  # 叉积 = ax*by - ay*bx
```

**浮点容差 `eps = 1e-10`**：计算机浮点运算有精度限制，实际共线的三点叉积可能
算出 `1.2e-15` 这样的极小非零值。引入容差判断"近似为零"，避免误判。

#### 批量分析：analyze_all()

```
输入: points = [(x1,y1), (x2,y2), ..., (xn,yn)]

对每个内部顶点 Pi（i = 1 到 n-2，不含首尾）:
    取 p1 = points[i-1]（前驱）
    取 p2 = points[i]  （当前拐点）
    取 p3 = points[i+1]（后继）
    调用 cross_product(p1, p2, p3)
    记录 index, cross_val, direction

输出: list of dict，每项包含 index / cross_val / direction
```

为什么首尾不算？因为第一个点没有"前驱"，最后一个点没有"后继"，无法构造向量。

#### 文件 I/O：load_points() / save_results()

- `load_points(filepath)` 读 CSV，跳过 `#` 注释行和无法解析为浮点数的表头
- `save_results(filepath, points)` 内部调用 `analyze_all()` 重新计算，写入带 BOM 的 UTF-8（Excel 兼容）

---

### 3.2 turning_plot.py — 绘图模块

#### 设计目标

把所有 Matplotlib 操作封装进 `CanvasWidget` 类，让主程序完全不需要操作 `ax`、`fig`、`canvas`。

#### CanvasWidget 的对外接口

| 接口 | 说明 |
|------|------|
| `__init__(parent)` | 创建 Figure/Axes，嵌入 Tkinter 父容器并自动 pack |
| `redraw(points, *, lines_visible, analysis_results, xlim, ylim)` | 清空画布，一次性绘制所有元素 |
| `connect(event_name, callback)` | 代理 `mpl_connect`，注册鼠标事件 |
| `ax` 属性 | 返回 Axes 对象，供事件处理中 `event.inaxes` 比较 |

#### redraw() 的绘制顺序

```
cla() 清空
  → 恢复坐标轴设置（xlim/ylim/aspect/grid）
  → 若 lines_visible: 画折线（zorder=1，最底层）
  → 画散点（zorder=2）
  → 画点号标注 P1,P2,... 和坐标标注（zorder=3）
  → 若 analysis_results 非空: 画拐向标注框（zorder=4，最顶层）
  → canvas.draw() 触发实际渲染
```

**zorder 图层控制**：数字越大越靠上。线(1) < 点(2) < 文字(3) < 分析标注(4)，
确保分析标注不被点或线遮住。

#### 关闭默认导航交互

```python
self.fig.canvas.toolbar_visible = False
self.fig.canvas.header_visible  = False
self._ax.set_navigate(False)
```

这三行关闭 Matplotlib 的默认缩放/平移行为，防止左键点击时出现选区框，
干扰我们自己实现的采点逻辑。

### 3.3 zxpd1.py — 主程序入口

#### 数据状态变量

| 变量 | 类型 | 含义 |
|------|------|------|
| `self.points` | `list[(x,y)]` | 所有采集的点，核心数据 |
| `self.analysis_results` | `list[dict]` | `analyze_all()` 的缓存结果 |
| `self.mode` | `str` | `'pick'` 采点模式 / `'drag'` 拖动模式 |
| `self.lines_visible` | `bool` | 是否绘制折线 |
| `self.analyzed` | `bool` | 是否已执行拐向分析 |
| `self.drag_index` | `int/None` | 当前拖动的点的索引 |
| `self.xlim / ylim` | `tuple/None` | 坐标轴范围（导入文件后自动设置） |

#### 统一重绘入口 _redraw()

```
self.canvas_widget.redraw(
    points           = self.points,
    lines_visible    = self.lines_visible,
    analysis_results = self.analysis_results（若 analyzed 为 False 则传 None）,
    xlim             = self.xlim,
    ylim             = self.ylim
)
```

所有操作（采点、撤销、拖动、导入、分析）最终都调用这一个方法，
保证画面状态和数据状态始终同步。

#### 采点与撤销

- **左键采点**：`self.points.append(...)` → 重置 `lines_visible` 和 `analyzed` → `_redraw()`
- **右键撤销**：`self.points.pop()` → 重置标志 → `_redraw()`
- 每次修改点后都要重置 `analyzed = False`，防止旧的分析结果与新的点不匹配

#### 拖动逻辑（三步配合）

```
鼠标按下（press）
  └─ 计算鼠标到所有点的欧氏距离（numpy 向量化）
  └─ 距离 < drag_threshold → 记录 drag_index（选中）

鼠标移动（motion）
  └─ drag_index 不为 None → 更新 points[drag_index] → _redraw()

鼠标释放（release）
  └─ drag_index = None（清除）
  └─ 若之前 analyzed: 自动调用 analyze_turning()（实时更新分析）
```

**向量化距离计算**：
```
distances = np.linalg.norm(pts - click, axis=1)
```
一次性算出鼠标到所有点的欧氏距离，比 for 循环快得多。

#### 业务方法分工

| 方法 | 调用的模块 | 说明 |
|------|-----------|------|
| `analyze_turning()` | `turning_algo.analyze_all()` | 计算所有拐向，缓存到 `analysis_results` |
| `_update_result_panel()` | 无（纯 Tkinter） | 把 `analysis_results` 写入右侧文本框 |
| `import_points()` | `turning_algo.load_points()` | 读文件，自动调整坐标轴范围 |
| `export_results()` | `turning_algo.save_results()` | 写文件 |

#### Tkinter Text 控件的写入流程

```
config(state=NORMAL)    ← 临时解锁（只读→可写）
delete('1.0', END)      ← 清空旧内容（'1.0' = 第1行第0列）
insert(END, text, tag)  ← 写入带颜色标签的文字
config(state=DISABLED)  ← 重新锁定（可写→只读）
```

三步缺一不可：DISABLED 状态下无法写入；写完后要锁回去，防止用户乱改结果。

## 6. 延伸：矩形绘制的算法原理

> 本节对应第 2 章后续内容（§2.6 判断矩形是否包含点 / §2.7 判断线段是否在矩形中等），
> 先建立"矩形"的几何表示基础，再讨论如何以交互方式绘制一个矩形。

---

### 6.1 矩形的两种表示方式

在 GIS 算法中，矩形有两种主要的表示形式，适用于不同的场景：

**方式一：轴对齐矩形（AABB）**

用四个数值定义：`(xmin, ymin, xmax, ymax)`

```
         xmin         xmax
    ymax  ┌───────────┐
          │           │
          │    AABB   │
          │           │
    ymin  └───────────┘
```

特点：
- 边总是平行于坐标轴
- 表示极简，只需 4 个数
- 判断点是否在内非常高效：只需两个不等式
- GIS 中常用作**最小外接矩形（MBR / MBB）**，是空间索引的核心数据结构

**方式二：任意方向矩形（四顶点法）**

用四个顶点 P₁, P₂, P₃, P₄ 按顺序（顺时针或逆时针）定义：

```
    P₁ ─────────── P₂
    │               │
    │               │
    P₄ ─────────── P₃
```

特点：
- 可以表示任意朝向的矩形
- 判断点是否在内需要向量叉积
- 适合表示旋转后的空间对象

---

### 6.2 交互绘制矩形的两种方法

#### 方法一：对角点法（仅适用于轴对齐矩形）

**操作步骤**：用户点击两个对角顶点 A 和 C。

```
用户点击 A(x₁, y₁) ──── 第一个角
用户点击 C(x₃, y₃) ──── 对角点（对面那个角）

自动推算另外两个角：
    B(x₃, y₁)  ← 与 A 同行，与 C 同列
    D(x₁, y₃)  ← 与 C 同行，与 A 同列
```

**几何示意**：

```
A(x₁,y₃) ─── B(x₃,y₃)       ← 其实应该是 ymax 在上
    │               │
D(x₁,y₁) ─── C(x₃,y₁)       ← ymin 在下
```

实际上，`ymax = max(y₁, y₃)`, `ymin = min(y₁, y₃)`，
因为用户可能从任意方向拖拽，不一定先点左上角。

这就是 GIS 软件中"画矩形"最常见的交互方式——本质是在确定一个 AABB。

---

#### 方法二：三点法（可绘制任意方向矩形）

**操作步骤**：用户依次点击三个点。

```
第1步：点击 P₁ ──── 矩形第一个角
第2步：点击 P₂ ──── 第一条边的另一端（确定方向和长度）
第3步：点击 P₃ ──── 确定宽度（P₃ 到直线 P₁P₂ 的距离就是矩形宽度）
自动推算：P₄ 使得 P₁P₂P₃P₄ 构成矩形
```

**推算 P₄ 的几何原理**：

矩形要求 P₁P₂ ∥ P₄P₃，且 P₁P₄ ∥ P₂P₃，所以：

```
向量 P₁P₂ = P₂ - P₁ = 方向向量 d
从 P₃ 出发，沿 -d 方向走同样距离，就是 P₄

P₄ = P₃ - (P₂ - P₁) = P₁ + P₃ - P₂
```

这实际上是利用了**平行四边形法则**：P₁、P₂、P₃、P₄ 满足 P₁+P₃ = P₂+P₄（对角线中点相同）。

---

### 6.3 矩形的几何性质与验证

给定四个顶点 P₁, P₂, P₃, P₄，如何验证它们构成的是矩形？

**条件一：所有内角都是直角**

相邻边的向量**点积**为零：

```
→        →           →        →
P₁P₂ · P₁P₄ = 0     （P₁ 处直角）
P₂P₃ · P₂P₁ = 0     （P₂ 处直角）
... 以此类推
```

点积 `a·b = aₓbₓ + a_yb_y`，值为零说明两向量垂直。

**条件二：对边平行且等长**

```
|P₁P₂| = |P₃P₄|     ← 对边等长
|P₂P₃| = |P₁P₄|     ← 另一对对边等长
```

**条件三（推论）：对角线互相平分**

```
(P₁ + P₃) / 2 = (P₂ + P₄) / 2     ← 两条对角线的中点重合
```

这三个条件中，满足"对角线互相平分 + 等长"就可以确定是矩形。

---

### 6.4 矩形与叉积的关系

回顾我们在 §2.3 学过的叉积：对矩形的四个顶点逐个应用叉积，
得到的结果有特殊意义。

**叉积结果全同号 → 凸多边形**

矩形是凸多边形，在每个顶点处向左转（或全部向右转，取决于顶点顺序），
所有叉积同号。这是凸多边形的判断依据（§2.4 的基础）。

**叉积绝对值 = 面积**

```
对于矩形的一个角 P₁→P₂→P₃（其中 P₂ 是直角）：

|cross| = |a||b|sin(90°) = |a||b| × 1 = 两边长度之积 = 该角的两边构成的矩形面积
```

由于夹角恰好是 90°，sin90° = 1，叉积的绝对值直接等于两边长度的乘积，
也就等于矩形的面积。

---

### 6.5 轴对齐矩形在 GIS 中的核心作用

学习矩形不仅仅是为了画矩形，更重要的是它在 GIS 空间查询中的基础地位：

**最小外接矩形（Minimum Bounding Rectangle, MBR）**

对任意几何对象（点集、折线、多边形），都可以求出包住它的最小轴对齐矩形：

```
xmin = min(所有点的 x)
xmax = max(所有点的 x)
ymin = min(所有点的 y)
ymax = max(所有点的 y)
```

MBR 是 GIS 空间索引（如 R 树）的基础数据结构。查询时先用 MBR 快速过滤，
排除明显不相交的对象，再对候选对象做精确判断。这是"粗过滤 + 精确判断"的经典两阶段策略。

**本章后续章节的逻辑链**：

```
§2.3 叉积判断拐向
    ↓  基础工具
§2.4 判断点是否在线段上（叉积 = 0 且坐标范围满足）
    ↓  组合使用
§2.5 判断两线段是否相交（叉积符号判断）
    ↓
§2.6 判断矩形是否包含点
    ←  AABB: xmin≤x≤xmax AND ymin≤y≤ymax（极简）
    ←  任意矩形: 对四条边分别用叉积判断点在同侧
        ↓
§2.7 判断线段/折线/多边形是否在矩形中
    = 端点在矩形内 AND 线段不与矩形边相交
```

### 6.6 如何在现有程序中扩展"绘制矩形"功能

在当前折线拐向程序的架构下，如果要新增"矩形绘制"模式，
按模块职责可以这样拆分：

**turning_algo.py 需要增加的算法**：
- `make_aabb(p1, p2)` — 从两个对角点构造轴对齐矩形的四个顶点
- `make_rect_3pt(p1, p2, p3)` — 从三点构造任意矩形的四个顶点
- `is_rect(points)` — 验证四点是否构成矩形（通过点积和边长）
- `rect_contains_point(rect_pts, p)` — 判断点是否在矩形内

**turning_plot.py 需要增加的绘图**：
- `CanvasWidget` 新增 `draw_rect(rect_pts, color)` 内部方法
- `redraw()` 增加 `rect_points` 参数，绘制矩形填充色和边框

**zxpd1.py 需要增加的 UI**：
- 工具栏新增 "🔲 矩形" 模式按钮
- 模式变量 `self.mode` 增加 `'rect'` 状态
- `_handle_pick()` 增加矩形模式的逻辑分支

这就是模块化设计的优势：**新功能只需要在对应的层级添加代码，不影响其他模块**。

## 7. 思考题

完成代码并阅读本节后，尝试思考以下问题：

1. **闭合折线的拐向**：如果折线是闭合的（首尾相连，即多边形边界），
   首尾两个点也变成了"内部顶点"，`analyze_all()` 该如何修改？
   - 提示：遍历范围从 `(1, n-1)` 改为 `(0, n)`，取点时用 `i % n` 取模

2. **浮点容差 `eps`**：`eps = 1e-10` 的取值对结果有什么影响？
   改成 `1e-2` 或 `1e-15` 分别会出现什么问题？
   - 提示：太大会误判"几乎共线但不完全共线"；太小会让浮点误差导致的微小值被判为有方向

3. **如何判断凸多边形**：利用叉积，如何判断一个多边形是不是凸多边形？
   - 提示：对所有顶点逐个计算叉积，若全部同号，则是凸多边形

4. **矩形面积计算**：已知矩形四个顶点（任意顺序），
   如何用叉积计算矩形面积，而不需要求边长？
   - 提示：叉积的绝对值等于以两向量为边的平行四边形面积；矩形面积 = 半个对角线叉积

5. **模块化设计的意义**：如果不做模块化，把所有代码写在一个 1000 行的文件里，
   新增"矩形"功能需要修改多少处？现在模块化之后需要修改多少处？各有什么风险？
